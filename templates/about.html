SQL PLAYGROUND Documentation

Discover how to transform natural language queries into precise SQL commands with our powerful AI-powered tool

Overview

The SQL Playground with NL to SQL Converter is an innovative web application designed to bridge the gap between natural language and database queries. It empowers users to interact with MySQL databases using plain English, eliminating the need to learn complex SQL syntax while providing powerful database exploration capabilities.

This comprehensive tool is perfect for:

  • Business analysts who need to query databases without SQL knowledge
  • Developers who want to quickly prototype and test database queries
  • Students learning both database concepts and natural language processing
  • Data professionals who need to quickly explore unfamiliar database structures
  • Anyone who wants to simplify database interactions using natural language

Features

Database Explorer

Browse all available tables in your MySQL database with a clean, intuitive interface. Instantly see your database structure and navigate between different tables with ease.

Schema Inspector

Examine detailed schema information for each table, including column names, data types, constraints, and primary keys. Understand your database structure at a glance.

Data Preview

View actual data from any table with a powerful, scrollable data grid. Explore your data without writing any SQL code, with support for large datasets.

Query Runner

Execute custom SQL queries with a built-in editor. View results instantly in a formatted data table with proper type formatting and easy navigation.

AI-Powered NL to SQL

Transform natural language questions into accurate SQL queries using advanced AI technology with the Mistral model. Supports complex queries with multiple conditions and intelligent date handling.

Connection Management

Easily configure and manage your MySQL database connections with an intuitive settings panel. Visual connection status indicator keeps you informed of your database connection state.

How to Use

Getting Started

When you first open the application, you'll need to configure your database connection. Click the settings icon in the top right corner and enter your MySQL database credentials. Once connected, you can explore all the features of the SQL Playground.

Configuring Database Connection

  1. Click the settings (gear) icon in the header
  2. Enter your MySQL host, username, password, and database name
  3. Click "Save Settings" to test the connection
  4. The connection status indicator will show green when successfully connected

Viewing Table Data

  1. Navigate to the "View Table Data" section
  2. Select a table from the dropdown menu (populated from your database)
  3. Click the "Load Data" button to retrieve records
  4. Scroll through the results to view all data
  5. Use horizontal scrolling if the table has many columns

Running Custom Queries

  1. Go to the "Run Custom Query" section
  2. Type or paste your SQL query into the text area
  3. Click "Execute Query" to run the command
  4. View the results in the formatted table below the editor
-- Example: Get customers from USA SELECT * FROM customers WHERE country = 'USA';

Natural Language to SQL Conversion

  1. Navigate to the "Natural Language to SQL" section
  2. Select the target table from the dropdown menu
  3. Type your question in plain English in the text area
  4. Click "Convert to SQL" to generate the query using AI
  5. Review the generated SQL code
  6. The results will automatically display below the SQL output
Pro Tip

The AI model intelligently handles date queries. If you mention a month or year, it will select the entire period. For example, "sales in January 2025" will return all records from that entire month.

Examples

Here are some practical examples of natural language queries and the SQL they generate. The AI model understands various query types including filtering, sorting, aggregation, and date operations:

Natural Language Input

"Show me all customers from California ordered by last name"

Generated SQL
SELECT * FROM customers WHERE state = 'California' ORDER BY last_name;
Natural Language Input

"Find products with price greater than $100 in the electronics category"

Generated SQL
SELECT * FROM products WHERE price > 100 AND category = 'electronics';
Natural Language Input

"Show total sales by month for 2024"

Generated SQL
SELECT MONTH(sale_date) AS month, SUM(amount) AS total_sales FROM sales WHERE YEAR(sale_date) = 2024 GROUP BY MONTH(sale_date);
Natural Language Input

"Customers who made purchases in the last 30 days"

Generated SQL
SELECT DISTINCT customers.* FROM customers JOIN orders ON customers.id = orders.customer_id WHERE orders.order_date >= DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY);
Tips for Better Results

Be specific in your natural language queries. Instead of "show me customers," try "show me customers from New York who signed up in the last month and have made more than 5 purchases." The more specific you are, the more accurate the generated SQL will be. Include table names, specific columns, filters, and sorting requirements for optimal results.
